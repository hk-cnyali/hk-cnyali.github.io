给出一个长度为 nnn 的字符串 sss 和 mmm 个区间 [li,ri][l_i, r_i][l​i​​,r​i​​], 给定kkk

qqq次 询问. 每次给定一个长度为 kkk 的字符串 www 以及一对 L,RL,RL,R, 求所有满足 i∈[L,R]i\in[L, R]i∈[L,R]的 w[li:ri]w[l_i:r_i]w[l​i​​:r​i​​] 在 sss 中的出现次数之和.

n,m,k,q≤1×105n,m,k,q\le 1\times 10^5n,m,k,q≤1×10​5​​, ∑∣w∣≤1×105\sum |w|\le 1\times 10^5∑∣w∣≤1×10​5​​

Solution

因为kkk一定，且∑∣w∣\sum|w|∑∣w∣不大，考虑根号平衡

先对sss串建SAM，设kq=Tkq = Tkq=T

  • 当k≤mk \le \sqrt mk≤√​m​​​ 时:

    因为kkk很小，所有可能的区间[li,ri][l_i, r_i][l​i​​,r​i​​]只有k2k^2k​2​​种，可以对每个询问暴力O(k2)O(k^2)O(k​2​​)枚举子串，丢到SAM上匹配求出对应right集合大小，再乘上这个子串对应的区间在[L,R][L, R][L,R]中的出现次数(用vector二分实现)即可

    复杂度是O(k2qlogn=O(Tmlogn)O(k^2q\log n = O(T\sqrt m\log n)O(k​2​​qlogn=O(T√​m​​​logn)

  • 当k>mk > \sqrt mk>√​m​​​时:

    此时询问很少，考虑对每个询问都暴力枚举[L,R][L, R][L,R]中的区间，然后倍增在SAM上匹配，复杂度是单词询问O(mlogn)O(m \log n)O(mlogn)

    要想每次倍增求的话首先要对www预处理出以每个点为右端点时，在SAM上匹配的节点，和最大匹配长度，这部分复杂度是单次询问O(n)O(n)O(n)的

    总复杂度O(q(n+mlogn))=O(nm+Tmlogn)O\big(q(n + m\log n)\big) = O(n\sqrt m + T\sqrt m\log n)O(q(n+mlogn))=O(n√​m​​​+T√​m​​​logn)
