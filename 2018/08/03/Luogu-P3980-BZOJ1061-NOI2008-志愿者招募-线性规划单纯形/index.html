「NOI2008」志愿者招募 - 线性规划单纯形

Posted at 18-8-3 16:35, Updated at 19-10-16 21:59

题目链接：传送门

Description

一个项目需要NNN天完成，第iii天至少需要AiA_iA​i​​ 个人。一共有MMM类志愿者可以招募，第iii类可以从第SiS_iS​i​​ 天工作到第TiT_iT​i​​天，招募费用是每人CiC_iC​i​​ 元。求满足条件的最小费用

Constraints

n≤1000,m≤10000n\leq 1000, m \leq 10000n≤1000,m≤10000，题目中其他所涉及的数据均不超过231−12^{31}-12​31​​−1

Solution

这道题原本是一道网络流建模神题，然而确是单纯形裸题

设第类志愿者招募了人，表示第天第类志愿者能否工作，把看做，看做，那么就是线性规划比较一般的形式了：

最小化                             ∑j=1ncjxj\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \sum_{j=1}^{n} c_jx_j                             ∑​j=1​n​​c​j​​x​j​​

满足约束                         ∑j=1nai,jxj≤bi               i=1,2,...m\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \sum_{j=1}^{n}a_{i,j}x_j \leq b_i\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ i=1,2,...m                         ∑​j=1​n​​a​i,j​​x​j​​≤b​i​​               i=1,2,...m

                                                        xj≥0                j=1,2,...n\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ x_j \ge 0\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ j=1,2,...n                                                        x​j​​≥0                j=1,2,...n

再对偶一下就变成了

最大化                             ∑i=1mbiyi\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \sum_{i=1}^{m} b_iy_i                             ∑​i=1​m​​b​i​​y​i​​

满足约束                         ∑i=1mai,jyi≤cj               j=1,2,...n\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \sum_{i=1}^{m}a_{i,j}y_i \leq c_j\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ j=1,2,...n                         ∑​i=1​m​​a​i,j​​y​i​​≤c​j​​               j=1,2,...n

                                                        yi≥0                i=1,2,...m\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ y_i \ge 0\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ i=1,2,...m                                                        y​i​​≥0                i=1,2,...m

然后直接上单纯形就可以了

单纯形通俗来讲就是通过不断地换元，代入消元来把目标函数∑cixi\sum{c_ix_i}∑c​i​​x​i​​的系数都变成负数，然后又因为xix_ix​i​​非负，就能在xix_ix​i​​都取000的时候求出最大值了
