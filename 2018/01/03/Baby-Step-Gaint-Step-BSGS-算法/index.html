Baby-Step-Gaint-Step(BSGS)算法

Posted at 18-1-3 15:13, Updated at 19-10-16 21:59

模板题：POJ2417 调这道题调了我一下午，第二天实在查不出错把#ifndef去掉就A了。。。。。

大致思想

BSGS这个算法主要是用来解决这个问题： Ax≡B(mod C)A^x\equiv B ( mod \ C)A​x​​≡B(mod C)已知ABC， 求x

而最普通的BSGS求解的是C为质数（其实就是相当与AC互质）的情况

对于这个问题，如果直接暴力枚举x的值的话，根据Fermat小定理可以知道Aϕp≡1(mod p)A^{\phi p} \equiv 1(mod\ p)A​ϕp​​≡1(mod p)

那么我们只要再往下枚举的话必定会产生循环

因此枚举的范围就是000~ϕC\phi CϕC，但是如果C特别大的话显然会超时，于是便有了BSGS算法

算法流程

我们设m=⌈C⌉m = \lceil \sqrt C \rceilm=⌈√​C​​​⌉（可以证明出只需要取到该值）,

令x=i∗m+jx = i * m + j x=i∗m+j 那么有Aim+j≡B(mod C)A^{im + j} \equiv B(mod\ C)A​im+j​​≡B(mod C)

因此Aim×Aj≡B(mod C)A^{im} \times A^j\equiv B(mod \ C)A​im​​×A​j​​≡B(mod C)

我们枚举i的话，就能得到AimA^{im}A​im​​的值

令D=AimD=A^{im}D=A​im​​，则 D×Aj≡(mod C)D \times A^j \equiv (mod \ C)D×A​j​​≡(mod C)

显然我们能用扩欧将AjA^jA​j​​求出，然后可以先预处理出所有AjA^jA​j​​的值，存在hash中

直接在hash中查找其所对应的j值即可（用map当然可以，只是会T。。。）

如果A和C不互质怎么办？

设d=gcd(A,C)d = \gcd(A, C)d=gcd(A,C)，则A=a∗d,B=b∗d,C=c∗dA=a * d, B = b * d, C = c* dA=a∗d,B=b∗d,C=c∗d(如果B不是d的倍数且B!=1的话显然无解)

也就是说(a∗d)x≡(b∗d)(mod c∗d)(a * d)^x \equiv (b*d) (mod \ c*d)(a∗d)​x​​≡(b∗d)(mod c∗d)

因此a∗(a∗d)x−1≡b(mod c)a * (a * d)^{x - 1} \equiv b(mod\ c)a∗(a∗d)​x−1​​≡b(mod c)

我们把之前的D∗=aD*= aD∗=a，那么前半部分就只剩下(a∗d)x−1(a * d)^{x - 1} (a∗d)​x−1​​了

所以我们不断求d=gcd(A,C)d = gcd(A, C)d=gcd(A,C),然后不断将B/=d,C/=d,D∗=A/dB /= d, C /= d, D *= A/dB/=d,C/=d,D∗=A/d，++cnt, 直到d==1为止

最后方程就变成这种形式了：

D∗Ax−cnt≡B(mod C)D * A^{x - cnt} \equiv B(mod\ C)D∗A​x−cnt​​≡B(mod C)

我们像之前一样令x=i∗m+j+cntx = i * m + j + cntx=i∗m+j+cnt，和之前一样处理即可

只是这样求出来的x≥cntx \ge cntx≥cnt，我们需要首先枚举一下0∼cnt−10\sim cnt-10∼cnt−1的情况就能解决了
