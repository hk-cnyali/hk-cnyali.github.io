先考虑树的情况

显然这个期望就是每种情况的概率和，又因为期望具有线性性，所以可以考虑两两点对对答案的贡献

有序点对(x,y)(x, y)(x,y)，树上路径长度为ccc（经过ccc个节点），对答案的贡献为1c\frac{1}{c}​c​​1​​

若在访问xxx时，yyy能够被访问，则说明xxx到yyy的这条链还没有断

也就是说，在所有删点方案中，只要xxx在x到y这条链除了x之外的任意一个点被删除之前删除，就会产生贡献

这个概率就是在只考虑x到y这条链的任意排列方案时，钦定第一个为xxx的概率，即为1c\frac{1}{c}​c​​1​​

所以我们只需要对每个ccc求出长度为ccc的路径条数

点分治 + NTT合并即可

这个用NTT合并的trick也不难理解

在点分治合并答案时，下标（其实就是深度）也是一个形如i+j=ki+j=ki+j=k的合并方式，且是乘法运算，所以可以用NTT优化

光以上的部分我就调了一年。。。

Base-Ring Tree

接下来考虑基环树，仍然是考虑点对(x,y)(x, y)(x,y)的贡献

  • (x,y)(x, y)(x,y)在同一棵子树内：与树的计算方法相同

  • (x,y)(x,y)(x,y)不在同一棵子树内：

    令dep[x]+dep[y]=cdep[x] + dep[y] = cdep[x]+dep[y]=c，则此时删除xxx时x,yx, yx,y仍联通的概率等于xxx在a+ca+ca+c个点中最先删除或在b+cb+cb+c个点中最先删除的概率（分别对应经过环的两侧）

    通过容斥我们可以得到此时贡献为1a+c+1b+c−1a+b+c\frac{1}{a + c} + \frac{1}{b + c} - \frac{1}{a + b + c}​a+c​​1​​+​b+c​​1​​−​a+b+c​​1​​

    随便选一条边破环为链，用类似CDQ分治的分治方法 + NTT即可

    其实这个分治方法也类似于序列上的点分治，就是每次考虑过中点的贡献

总复杂度O(nlog2n)O(n\log^2 n)O(nlog​2​​n)

具体实现上，可以先破环为链，对剩下的部分计算一次树情况的答案

这部分我们算出来的答案既包括了每个子树的答案，又包括了各个子树之间通过环上某一条路径的答案

例如，假设破的环在aaa部分中，那么我们相当于已经计算了1b+c\frac{1}{b+c}​b+c​​1​​的答案了

那么我们只需要分治计算1a+c\frac{1}{a+c}​a+c​​1​​和1a+b+c\frac{1}{a+b+c}​a+b+c​​1​​的贡献了

这样代码写起来会比较方便，更具体的部分可以直接看代码了，自我感觉写得应该是比较容易理解的
