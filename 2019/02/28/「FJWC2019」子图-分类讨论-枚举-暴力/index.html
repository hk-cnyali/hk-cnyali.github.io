Pre-Algorithm

你得会在O(mm)O(m\sqrt m)O(m√​m​​​)的复杂度内进行三元环和四元环计数

Three-Membered Ring

首先把无向图转成有向图，由度数小的向度数大的连边，度数相同就由编号小的向编号大的

那么这样连出来的一定是一个有向无环图

然后考虑一个暴力做法：

  • 先枚举点iii， 把所有iii连向的点标记为iii
  • 枚举iii连向的点jjj
  • 枚举jjj连向的点kkk，若kkk被标记为iii，则存在一个三元环(i,j,k)(i, j, k)(i,j,k)

因为无向图被我们转化成了dag，所以每个三元环只会在度数最小的点处被统计，不会算重

复杂度的话，显然等于所有被指向点的出度的和

而每个点的出度是不会大于m\sqrt m√​m​​​（若存在一个点出度大于m\sqrt m√​m​​​，那么它所有出边指向的点的度数都要大于m\sqrt m√​m​​​，边的总数就会大于mmm）

所以复杂度为O(mm)O(m\sqrt m)O(m√​m​​​)

Four-Membered Ring

四元环的计数与三元环类似，先枚举点iii，然后枚举连向的点jjj，再枚举jjj连向的点kkk。记录一个cntcntcnt数组，答案每次加上cnt[k]cnt[k]cnt[k]，然后++cnt[k]。但是在这里不能转化成有向图

因为如果建成有向图，设valvalval表示一个点按(deg,id)(deg, id)(deg,id)排序的值，那么有vali<valj,vali<valkval_i < val_j, val_i < val_kval​i​​<val​j​​,val​i​​<val​k​​，但是我们并不确定valjval_jval​j​​和valkval_kval​k​​的大小关系

在三元环计数中valjval_jval​j​​和valkval_kval​k​​的大小关系可以任意，而四元环中必须要满足valj<valkval_j < val_kval​j​​<val​k​​，所以不行

考虑一个类似的做法，将所有点重标号，并且把每个点连出的边按照所连出点的valvalval值排序

在枚举点的过程中，始终保证vali<valj,vali<valkval_i < val_j,val_i < val_kval​i​​<val​j​​,val​i​​<val​k​​，如果不满足就立刻break

与三元环证明方法类似，这个做法也可以被证明是O(mm)O(m\sqrt m)O(m√​m​​​)的

Solution

大力枚举 + 分类讨论

k=3k=3k=3时，讨论一下菊花、链、三元环的情况即可，其中算链的时候会多算三次三元环的情况，注意要减去

下面讨论k=4k=4k=4的5种情况：

  1. 菊花：

    枚举点iii，用度数算一下

  2. 一条边，一边连两个点，另一边连一个点：

    枚举边iii，用度数算一下。此时ccc可能与aaa或bbb重合形成情况3，所以要减掉222次情况3

  3. 三元环，连一个点：

    枚举三元环，用三元环上点的度数算一下

  4. 链：

    枚举点iii，枚举与iii相邻的点yyy，aaa为当前的yyy，ccc为之前枚举过的yyy，然后利用度数计算贡献，此时多算的情况：

    • bbb和ddd重合形成情况5
    • bbb和ccc重合且aaa和ddd重合，形成情况3
    • aaa和ddd重合且bbb和ccc重合，形成三元环

    总共要减去222次情况5，222次情况3，111次三元环

  5. 四元环：

    之前讲过了
