  • 1, _x, _y

    令x=(_x+last−1)mod n+1,y=(_y+last−1)mod n+1x = (\_x + last - 1) \mathrm{mod}~n + 1, y=(\_y + last - 1) \mathrm{mod}~n + 1x=(_x+last−1)mod n+1,y=(_y+last−1)mod n+1

    若(x,y)(x, y)(x,y)已经连接，则删掉(x,y)(x, y)(x,y)，否则连接(x,y)(x, y)(x,y)

  • 2, _x, _y

    令x=(_x+last−1)mod n+1,y=(_y+last−1)mod n+1x = (\_x + last - 1) \mathrm{mod}~n + 1, y=(\_y + last - 1) \mathrm{mod}~n + 1x=(_x+last−1)mod n+1,y=(_y+last−1)mod n+1

    询问xxx和yyy是否联通，输出0/10/10/1

Solution

看上去是一道动态图板子

但实际上因为这里的答案只可能是0/10/10/1，可能的操作只有两种，所以可以离线线段树分治搞

具体来说，对于一次修改，把两种可能的操作都先搞出来。对于一个操作(x,y)(x, y)(x,y)，处理出它后面第一个可能的操作时间nxtnxtnxt

线段树分治时，维护一个全局的Sx,yS_{x, y}S​x,y​​，表示边(x,y)(x, y)(x,y)当前的状态（已经连上/没连上）

线段树处理到l ==r的时候，我们是知道当前的答案的，于是把答案对应的那一种操作的SSS异或上111。再对两种操作看其SSS是否为111，若为111，则在分治的线段树中，把当前操作应用到[l+1,nxt][l + 1, nxt][l+1,nxt]区间上

感觉没讲清楚。。。不过脑补一下并不难

Summary

这个离线做法本质上是把所有可能的情况都处理出来，把操作拆成若干段，只有跨过段的时候这条边的影响才有可能变化

感觉是很巧妙的一道题
