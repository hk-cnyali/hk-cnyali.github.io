反悔型贪心

似乎就是就是模拟费用流？

CF867E

Description

你预言了每天的股票价格 viv_iv​i​​ , 从第 111 天开始,你每天可以选择卖一支股票,买一支股票,或者什么也不做

问直到第 nnn 天结束你最多可以获得多少收益

n≤3×105n \le 3 \times 10^5n≤3×10​5​​

Solution

定义一次股票买卖(x, y)为，把xxx买进，yyy卖出

考虑贪心，维护一个小根堆，每次将当前股票价格扔进去

对于当前股票xxx而言，看堆里最小的元素yyy是否比它小，如果是，则做一次股票买卖(y,x)(y, x)(y,x)，答案加上x−yx - yx−y

显然，这样做会有问题，因为有可能yyy在后面的某个时刻卖出更优

如何反悔？

依旧是上面的做法，把yyy弹出堆后，将xxx扔进堆里两次即可

这两次的意义是不一样的，分别是这两个意义：

  • 把xxx作为后面某一次股票买卖的买进部分
  • 把xxx看成中转站，可以理解为，xxx仍旧是后面某次股票买卖(x,z)(x, z)(x,z)的买进部分，不过在现在xxx已经被卖出了。一卖一买相互抵消，就相当于进行了一次(y,z)(y, z)(y,z)的买卖，答案会加上x−y+z−x=z−yx - y + z - x = z - yx−y+z−x=z−y

这样就实现了贪心过程中的反悔

CF967E

其实之前写过，但是感觉写得不太清楚

Description

给你一个数列AAA，求是否存在它的一个排列使得前缀异或和单调递增。如果存在，则输出一种方案

n≤105,Ai≤260n\le 10^5, A_i\le 2^{60}n≤10​5​​,A​i​​≤2​60​​

Solution

所有满足 s⊕k>ss \oplus k > ss⊕k>s 的 kkk 必然满足, kkk的最高位在 sss 上为 0.

于是把所有数按二进制最高位存起来, 每次对当前前缀和从低到高 chk 每个为 000 的位，是否是某个未选数最高位的落点.

如果有, 用它更新当前前缀和, 再去填下一个数, 重新找. 如果扫到最高位都没有, 就无解

显然从低位往高位贪心更优，因为选择会更多。若从高到低最多只能放606060个数

另外，对于最高位相同的数，选择的先后顺序并没有影响
